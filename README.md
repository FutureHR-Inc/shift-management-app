## 要件定義

### 1. 概要
3店舗（京橋・天満・本町）に勤務する社員・アルバイトのシフト作成および管理をWebアプリケーション上で効率化・汎用化することを目的とする。

### 2. 対象ユーザー
- 店舗責任者（店長）
- アルバイト・社員スタッフ

### 3. 主要機能要件

#### 4.1 シフト作成・管理機能（店長）
- カレンダーをクリックしてシフトを入力・編集
- シフトパターン（通し、ランチ等）をテンプレートとして選択可能
- スタッフ名を選択することで、該当スタッフのスケジュールを自動反映
- 店舗ごとに設定した必要人数と実人数をリアルタイムで比較表示（赤：不足／青：過剰）
- 「確定」操作後、勤務時間・残業時間・人件費概算を自動計算

#### 4.2 スタッフ希望申請・参照機能（スタッフ）
- スマホ／PCブラウザでログイン、操作可能（レスポンシブ）
- 自分のシフトがカレンダーに表示される
- 希望休申請が可能（店長に通知）
- 希望休は店長がワンクリックで承認・却下
- シフトが確定すると、当日0:00に「今日のシフト」が自動プッシュ通知

#### 4.3 欠員対応機能
- 急な欠勤が出た場合、「代打募集」ボタンで全スタッフへ一斉通知
- 出勤可能なスタッフは「参加」ボタンで即応答可

#### 4.4 勤怠ルール管理
- 任意の勤怠ルール（例：週28時間超、7連勤）を設定可能
- 違反時に自動警告表示

#### 4.5 店舗設定機能
- 各店舗ごとに曜日別・時間帯別の必要人数を設定
- 「回せる人（応援に行けるスタッフ）」を指定可能
- 「固定出勤（例：社長は毎週月曜ランチ）」の自動反映設定が可能

#### 4.6 表示・視認性向上
- シフト種別ごとに色分け表示（例：通し＝オレンジ、ランチ＝水色など）

#### 4.7 認証機能およびユーザー属性
- IDおよびパスワードによるログイン認証
- ユーザー属性に応じた操作可能範囲の制御
- 店舗責任者：自店舗のシフト作成・編集・承認、希望休承認、スタッフ管理、代打通知発信
- スタッフ・アルバイト：自分のシフト閲覧、希望休申請、代打応募

### 4. シフト作成操作フロー（店長）
1. ブラウザでURLアクセスし、ログイン
2. 「店舗設定」から各店舗の必要人数や回せる人を設定
3. 「スタッフ管理」でスタッフ情報を登録
4. 「シフトパターン」を登録（最大25件）
5. 週表示または月表示を選び、空きマスをクリックしシフトを登録
6. 全体の人数過不足を色で確認し、必要に応じて調整
7. シフトを「確定」して通知・計算を実行

## 📧 メール送信機能

### 自動メール送信一覧

#### 1. 🌅 毎日のシフト通知（定期実行）
- **タイミング**: 毎日午前0:00（JST）
- **対象**: 今日シフトがあるスタッフ
- **内容**: 今日のシフト詳細
- **実行**: `/api/cron/daily-shift-notifications`

#### 2. ✅ シフト確定通知
- **タイミング**: シフトが確定された時
- **対象**: 該当スタッフ
- **内容**: 確定したシフトの詳細一覧
- **実行**: 手動（管理者）

#### 3. 🚨 代打募集通知
- **タイミング**: **スタッフが代打募集を作成した時**
- **対象**: 該当店舗の関連スタッフ（募集者以外）
- **内容**: 代打募集の詳細 + 応募リンク
- **実行**: `/api/emergency-requests` (POST) で自動送信

#### 4. 🎉 代打採用通知
- **タイミング**: 管理者が代打を承認した時
- **対象**: 採用されたスタッフ + 元のスタッフ
- **内容**: 採用通知 + シフト詳細
- **実行**: `/api/emergency-requests` (PATCH) で自動送信

#### 5. 📋 シフト希望提出確認
- **タイミング**: スタッフがシフト希望を提出した時
- **対象**: 提出したスタッフ
- **内容**: 提出確認 + 提出内容概要
- **実行**: `/api/shift-requests` (POST) で自動送信

#### 6. ⏰ シフト希望提出締切リマインダー
- **タイミング**: 締切1日前（未提出者のみ）
- **対象**: シフト希望未提出のスタッフ
- **内容**: 締切リマインダー + 提出リンク
- **実行**: `/api/cron/shift-request-reminders` (毎日実行)

#### 7. 📢 一般通知
- **タイミング**: 管理者が任意で送信
- **対象**: 指定されたユーザー
- **内容**: カスタムメッセージ
- **実行**: 手動（管理者）

### メール送信の仕組み

```typescript
// メール送信フロー
1. イベント発生 → API処理
2. API内で `/api/email` にPOST
3. Resend APIで実際のメール配信
4. 受信者にメール到着
```

### クーロンジョブ設定

#### 毎日のシフト通知
- **URL**: `/api/cron/daily-shift-notifications`
- **実行時間**: 毎日 0:00 JST

#### 締切リマインダー
- **URL**: `/api/cron/shift-request-reminders`
- **実行時間**: 毎日 10:00 JST（締切1日前の未提出者に送信）

### 設定

メール送信にはResend APIを使用しています。

```env
RESEND_API_KEY=your_resend_api_key
NEXT_PUBLIC_APP_URL=https://your-app-url.com
CRON_SECRET=your_cron_secret (optional)
```
